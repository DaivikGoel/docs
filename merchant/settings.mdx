## Wallet Management

### Primary Wallet

Your primary wallet receives all payments:

1. **Current Wallet**: View connected wallet address
2. **Change Wallet**: Update to a different address
3. **Wallet Type**: MetaMask, WalletConnect, etc.
4. **Network**: Ensure Base network is selected

<Warning>
Changing your wallet address will only affect future payment links. Existing links will continue using their assigned contracts.
</Warning>

### Smart Contract Settings

Configure your payout splitter contracts:

<AccordionGroup>
  <Accordion title="Single Recipient">
    Default setup where all funds go to one wallet:
    - 100% to primary wallet
    - No splits required
    - Simplest configuration
  </Accordion>
  <Accordion title="Multiple Recipients">
    Split payments between multiple wallets:
    - Add recipient addresses
    - Set percentage for each
    - Must total 100%
    - Automatic distribution
  </Accordion>
  <Accordion title="Fee Recipients">
    Separate wallets for fees:
    - Platform fee wallet
    - Referral fee wallet
    - Charity donation wallet
    - Custom fee structures
  </Accordion>
</AccordionGroup>

## Payment Preferences

### Fee Configuration

Choose your default fee handling:

| Setting | Customer Pays | You Receive | Best For |
|---------|--------------|-------------|----------|
| **Absorb Fees** | Exact amount | Amount - fees | Clean pricing |
| **Pass Fees** | Amount + fees | Full amount | Margin preservation |

### Default Settings

Set defaults for new payment links:
- **Currency**: USD (USDC settlement)
- **Fee Mode**: Absorb or pass to customer
- **Private Matching**: On/off by default
- **Expiration**: Default link lifetime

### Webhook Configuration

Set up automated notifications:

```json
{
  "webhook_url": "https://your-site.com/webhook",
  "events": [
    "payment.completed",
    "payment.failed",
    "link.created",
    "link.updated"
  ],
  "secret": "your-webhook-secret"
}
```

## Backup and Export

### Data Export

Export all your data:
1. Click "Export Account Data"
2. Select data types:
   - Payment links
   - Transactions
   - Customer data
   - Settings
3. Choose format (CSV/JSON)
4. Download securely

### Account Backup

Regular backups include:
- Configuration settings
- Payment link templates
- Integration settings
- Custom scripts

<Tip>
Set up automatic weekly backups to ensure you never lose important configurations.
</Tip>

## Danger Zone

### Critical Actions

<Warning>
These actions cannot be undone. Proceed with caution.
</Warning>

- **Close Account**: Permanently delete account
- **Clear All Data**: Remove all transactions
- **Reset Settings**: Restore defaults
- **Revoke All Keys**: Invalidate API access

### Settings FAQ

<FAQ>
  <FAQItem question="Can I change my wallet address?">
    Yes, but it only affects new payment links. Existing links continue using their assigned contracts.
  </FAQItem>
  <FAQItem question="How do I enable 2FA?">
    Go to Security settings, click "Enable 2FA", and follow the QR code setup with your authenticator app.
  </FAQItem>
  <FAQItem question="What happens to my data if I close my account?">
    Transaction records are retained for legal compliance, but your account access is terminated.
  </FAQItem>
</FAQ>
