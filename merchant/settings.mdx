## Settings Overview

The settings page currently allows you to configure webhooks to receive automated notifications when transactions are completed.

<Info>
Additional settings and configuration options are planned for future releases.
</Info>

## Webhook Configuration

Webhooks allow you to receive real-time notifications when payments are completed.

### Setting Up Your Webhook

<Steps>
  <Step title="Navigate to Settings">
    Click on "Settings" in your dashboard navigation
  </Step>
  
  <Step title="Go to Webhook Tab">
    Select the "Webhook" tab in the settings page
  </Step>
  
  <Step title="Enter Your URL">
    Input your HTTPS webhook endpoint URL
    ```
    https://your-site.com/api/webhook
    ```
  </Step>
  
  <Step title="Save Configuration">
    Click "Save Webhook" to activate
  </Step>
  
  <Step title="Test Your Webhook">
    Use the "Test Webhook" button to verify your endpoint receives test payloads
  </Step>
</Steps>

### Webhook Payload

When a transaction is completed, you'll receive:

```json
{
  "event": "transaction.completed",
  "orderId": "ORD-ABC123-XYZ789",
  "transactionHash": "0x...",
  "status": "COMPLETED",
  "merchantAmount": 94.00,
  "feeAmount": 6.00,
  "totalAmount": 100.00,
  "timestamp": "2024-01-01T12:00:00.000Z",
  "paymentLinkId": "clqabc123xyz",
  "paymentLinkTitle": "Product Name",
  "recipientEmail": "customer@example.com",
  "paymentMethod": "BANK_TRANSFER",
  "currency": "USD"
}
```

### Webhook Requirements

- **HTTPS Only**: Your endpoint must use HTTPS
- **Response Time**: Respond with 200 OK within 10 seconds
- **Content Type**: Expects `application/json`

<Note>
Currently, only `transaction.completed` events are sent. Additional event types are planned for future releases.
</Note>

For detailed webhook implementation, see our [Webhook Documentation](/merchant/webhooks).
</Info>

### Settings FAQ

<FAQ>
  <FAQItem question="Why is my webhook not receiving events?">
    Ensure your endpoint uses HTTPS and responds with 200 OK within 10 seconds. Use the "Test Webhook" button to debug.
  </FAQItem>
  <FAQItem question="Can I have multiple webhook URLs?">
    Currently, only one webhook URL is supported per account.
  </FAQItem>
  <FAQItem question="What events can I receive?">
    Currently, only `transaction.completed` events are sent when payments are successfully processed.
  </FAQItem>
</FAQ>
