## Transaction Dashboard

The Transactions page provides a comprehensive view of all payments processed through your Shor account. Access it from the main navigation menu.

## Transaction Table Overview

### Table Columns

| Column | Description | Sortable |
|--------|-------------|----------|
| **Transaction ID** | Unique identifier for each payment | Yes |
| **Customer** | Wallet address or customer identifier | No |
| **Amount** | Original payment amount in USD | Yes |
| **USDC Received** | Actual USDC amount after conversions | Yes |
| **Date & Time** | Transaction timestamp (UTC) | Yes |
| **Status** | Current transaction state | Yes |
| **Payment Link** | Which link was used | Yes |
| **Payout Wallet** | Destination wallet address | No |
| **Transaction Hash** | Blockchain verification link | No |

### Understanding Transaction Data

<Tabs>
  <Tab title="Transaction ID">
    - Unique identifier for internal tracking
    - Format: `ORD-XXXX-XXXX-XXXX`
    - Use for support queries
    - Searchable in dashboard
  </Tab>
  <Tab title="Customer Info">
    - Shows wallet address or email
    - Click to copy address
    - May include payment method
    - Helps identify repeat customers
  </Tab>
  <Tab title="Amounts">
    **Amount**: What customer paid
    **USDC Received**: What you received
    - May differ due to:
      - Conversion rates
      - Network fees
      - Platform fees
  </Tab>
  <Tab title="Blockchain Data">
    - **Transaction Hash**: Proof of payment
    - **Payout Wallet**: Your receiving address
    - Click to view on Basescan
    - Immutable record
  </Tab>
</Tabs>

## Filtering and Searching

### Quick Filters

Use preset filters for common queries:
- **Today**: Transactions from last 24 hours
- **This Week**: Current week's transactions
- **This Month**: Current month's activity
- **All Time**: Complete history

### Advanced Filtering

<AccordionGroup>
  <Accordion title="By Date Range">

    3. Apply filter
    4. Export results if needed
  </Accordion>
  <Accordion title="By Status">

    - Show only completed
    - Show only pending
    - Show failed transactions
    - Combine multiple statuses
  </Accordion>
  <Accordion title="By Payment Link">

## Reconciliation

### Matching Payments

Steps for reconciliation:
1. Export transactions for period
2. Compare with bank/wallet statements
3. Match USDC deposits
4. Identify any discrepancies
5. Verify on blockchain if needed

### Common Reconciliation Issues

<Warning>
Always verify large or unusual transactions on the blockchain using the transaction hash.
</Warning>

Issues and solutions:
- **Missing transactions**: Check pending status
- **Amount differences**: Review fee settings
- **Timing mismatches**: Consider timezone differences
- **Duplicate entries**: Verify transaction hashes

## Customer Support

### Helping Customers

When customers inquire about payments:
1. Ask for transaction ID or amount/date
2. Search in your dashboard
3. Verify transaction status
4. Provide transaction hash if requested
5. Explain any delays or issues

### Common Customer Questions

<FAQ>
  <FAQItem question="Where is my payment confirmation?">
    Provide the transaction ID and hash. Show them how to verify on Basescan.
  </FAQItem>
  <FAQItem question="Why is my payment pending?">
    Explain blockchain confirmation times and that it typically resolves in 2-5 minutes.
  </FAQItem>
  <FAQItem question="Can I get a refund?">
    Explain that blockchain transactions are irreversible, but you can process refunds manually if your policy allows.
  </FAQItem>
</FAQ>
