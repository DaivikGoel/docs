## Request

<ParamField query="page" type="integer" default="1">
  Page number for pagination.
</ParamField>

<ParamField query="limit" type="integer" default="20">
  Number of items per page. Maximum 100.
</ParamField>

<ParamField query="status" type="string">
  Filter by status.
  
  **Options**:
  - `active`: Only active links
  - `inactive`: Only inactive links
  - `all`: All links (default)
</ParamField>

<ParamField query="sort" type="string" default="created_desc">
  Sort order for results.
  
  **Options**:
  - `created_desc`: Newest first (default)
  - `created_asc`: Oldest first
  - `amount_desc`: Highest amount first
  - `amount_asc`: Lowest amount first
  - `title_asc`: Alphabetical by title
</ParamField>

<ParamField query="search" type="string">
  Search payment links by title or description.
</ParamField>

<ParamField query="created_after" type="string">
  Filter links created after this date (ISO 8601).
  
  **Example**: "2024-01-01T00:00:00Z"
</ParamField>

<ParamField query="created_before" type="string">
  Filter links created before this date (ISO 8601).
  
  **Example**: "2024-12-31T23:59:59Z"
</ParamField>


## Pagination

Handle pagination in your application:

```javascript
async function getAllPaymentLinks() {
  let allLinks = [];
  let page = 1;
  let hasMore = true;
  
  while (hasMore) {
    const response = await fetch(
      `https://api.shorpay.com/api/v1/merchant/payment-links/get-payment-links?page=${page}&limit=100`,
      {
        headers: { 'Authorization': 'Bearer YOUR_API_KEY' }
      }
    );
    
    const data = await response.json();
    allLinks = allLinks.concat(data.data.paymentLinks);
    
    hasMore = data.data.pagination.hasNext;
    page++;
  }
  
  return allLinks;
}
```

## Performance Tips

1. **Use pagination** - Don't request all links at once
2. **Cache results** - Payment link data doesn't change frequently
3. **Filter server-side** - Use query parameters instead of client filtering
4. **Request only needed fields** - Use field selection (coming soon)

## Rate Limits

- Standard: 1000 requests per hour
- Returns all links are counted as single request regardless of pagination
